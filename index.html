Software / Libraries

General:

  • Code from the Open Energy Monitor project, who use Nanode RF's

Ethernet:

  • Jeelabs EtherCard Library (Nanode 5 & RF)
  • If you are using Arduino 0022 or 0023, you should use this snapshot of EtherCard. This version will not work on Arduino 1.0.
  • If you are using Arduino 1.0, you should use the new version of EtherCard. This version will not work on versions older than 1.0.
  • Blog post about using the EtherCard library with Nanode - includes uploading to Pachube (Nanode 5 & RF)
  • Andrew Lindsay's EtherCard Examples (Nanode 5 & RF)
  • Code to read the MAC address from the 11AA02E48 (Nanode 5 & RF)
  • Explaination of the different versions of the MCP7941x (Nanode RF)
  • Ian's Code to read the MAC address from the MCP79411 (Nanode RF)
  • Ian's code to read the MAC address from the 11AA02E48 and write it to a blank MCP79410 (Nanode RF)
  • Andrew Lindsay's EtherShield Library and wiki - includes examples for DHCP, DNS, Web Page, Pachube etc (Nanode 5 & RF) - Note: as of Nov 2011, Andy is no longer maintaining the EtherShield library and is contributing to EtherCard which is now the recommended library to use.
  • Andrew Lindsay's blog post on migrating from the EtherShield to EtherCard library (Nanode 5 & RF)
  • Code to get and display the time from an NTP server using EtherShield (Nanode 5 & RF)

Wireless / RF:

  • Information about the RFM12B RF modules used on the Nanode RF & Wi-Node
  • Jeelabs RF12 Library for using the RFM12B - be sure to read through the blog posts it links to on at the bottom of the page (Nanode RF & Wi-Node)
  • Send and receive sketches to test wireless functionality between two boards (Nanode RF & Wi-Node)
  • If you are using Arduino 0022 or 0023, you should use this ports library and this RF12 library. This will not work on Arduino 1.0.
  • If you are using Arduino 1.0, you should use the latest jeelib which is the Ports and RF12 libraries combined. This version will not work on versions earlier than 1.0.
